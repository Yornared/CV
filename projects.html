Wizzard Chess: Automatic Chessboard

This project is a fully automated chessboard that physically moves pieces using a CoreXY gantry system, stepper motors, and an electromagnet. The board communicates wirelessly with an ESP32-S3 microcontroller, which interprets moves from the Lichess API and translates them into precise motor commands.

  • Design: Built on a cost-effective CoreXY rail system, inspired by 3D printer mechanics for accuracy and simplicity.
  • Electronics: Powered by stepper motor drivers, an electromagnet for moving pieces, and a custom-built chess clock with wireless communication via nRF24.
  • Software: A React-based web app connects the board to Lichess, tracks the game state, and generates movement scripts for the magnet. Special handling was implemented for knight jumps, castling, and captures.

Features:

  • Play against AI or human opponents on Lichess while the board mirrors the moves in real time.
  • Accurate calibration with corner sensors and automatic re-homing.
  • Bluetooth connection for setup and control.

My contributions: I programmed the motor control system on the ESP32-S3, implemented Bluetooth Low Energy and webserver communication with the React app, and co-designed and assembled the CoreXY rail system and board hardware.

UniGigs: Gamified Freelancing SaaS for Universities

UniGigs is a Software-as-a-Service (SaaS) platform that enables freelancing services within university communities. It integrates gamification elements such as token-based transactions, login streak rewards, user levels, and leaderboards to drive engagement.

Design & Features:
• Dual-role system for service providers and seekers.
• Token-based currency with real money conversion.
• Gamification: login streaks, user levels, leaderboards.
• Real-time chat and notifications.
• Profile management and gig tracking.

Development Context:
Built by a team of 5 members over 12 weeks. The platform uses a React + TypeScript frontend, Firebase backend, and Stripe integration for payments.

My Contributions:
• Set up and managed the Firebase NoSQL database (initially planned with Dataconnect, but migrated due to beta limitations).
• Implemented the My Posted Gigs and Schedule tabs for managing gigs.
• Performed design cleanup to ensure consistency across views.
• Actively contributed to brainstorming features and shaping the overall design direction.
